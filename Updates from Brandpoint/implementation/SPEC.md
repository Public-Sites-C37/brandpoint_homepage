# SPEC.md — Brandpoint Homepage Update v2

## Problem Statement

Brandpoint has decided to promote the product landing page (currently at GitHub Pages preview) to become the **brandpoint.com homepage**. MaDonna Sheehy provided 16 change requests via email and Teams recording on 2026-03-30, including 4 new sections (sticky nav, value prop, publisher reel, footer), 6 asset swaps (real logos, real images replacing placeholders), and 4 text/content edits. All changes must be implemented in a single pass while maintaining the existing design language, responsiveness, and WordPress compatibility.

## Expected Behavior

After implementation, the page will:
- Have a sticky top navigation with Brandpoint logo and 5 links (Who We Are, Who We Work With, Blog, Editorial Content Library, Contact Us)
- Display the provided `Distribution Map-Dots.png` in the hero instead of the SVG map
- Include a new value prop section (lorem ipsum) between hero and social proof
- Show 10 real PR agency logos in the social proof slider
- Display distribution packages WITHOUT prices and WITHOUT "Most Popular" badge
- Use the `BPTOPTIMIZE326BLU.png` logo image in the "Meet Brandpoint Optimize" header
- Include a new quote placeholder before the AI Visibility section
- Show "Generated by Brandpoint Intelligence proprietary formula" as the gauge sub-header
- Display 3 real images in Campaign Customizations (AP/USA Today/Business Journals logos, Specialty Formats composite, Native Ads composite)
- Include a Premium Publisher Network logo reel with 16 outlet logos
- Show the CooperVision case study in a 2-column layout with image on the left
- Have a full site footer matching brandpoint.com's footer content in the new design language
- Pass SEO review (updated meta tags, alt text, heading hierarchy)

## Observed Behavior

The page currently has placeholder logos, SVG map, prices displayed, CSS text treatment for Optimize header, centered-text-only case study, and no nav/footer.

## Constraints

1. **Single pass implementation** — all 16 changes in one commit
2. **No build tools** — vanilla HTML/CSS/JS only
3. **WordPress pasteable** — all CSS classes namespaced `bp-landing-`
4. **Placeholder text** — value prop section and new quote use lorem ipsum; clearly marked for MaDonna to replace
5. **Absolute URLs** — all nav/footer links use full `https://www.brandpoint.com/...` URLs
6. **Asset filenames** — normalize to lowercase-hyphenated when copying to `assets/`
7. **No Boston Herald** — intentionally omitted from publisher reel and map per provided assets
8. **Brandpoint logo** — needs to be sourced for nav/footer (not yet in assets folder)
9. **Image sizes** — use provided PNGs as-is; image optimization is a future production task
10. **Filename mapping** — source files are normalized to semantic lowercase-hyphenated names:
    - `Distribution Map-Dots.png` → `distribution-map.png`
    - `BPTOPTIMIZE326BLU.png` → `bpt-optimize-logo.png`
